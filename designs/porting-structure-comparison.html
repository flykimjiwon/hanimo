6. 왜 이렇게 빠른가 — 3가지 이유

이유근거
🪞 양쪽 CLI가 거의 같음internal/* 16 vs 15 디렉토리, 그중 15개 동명. 파일 이름 95% 일치. 이미 여러 차례 양방향 포팅 해왔음 (sync-tracker 참조).
🔧 IDE는 단일 Wails 앱techai-ide = 단일 Go 모듈 (2,813 LOC) + 단일 React 앱 (3,644 LOC). monorepo 없음. 복사 후 sed만 통과하면 빌드 성공.
🎨 Honey 테마는 CSS 변수이미 designs/hanimo-desktop-v1.html에 완성된 CSS 블록 있음 (54 lines). 복붙 + Picker 한 줄 + App.tsx 한 줄.

7. 시간이 더 걸릴 수 있는 유일한 리스크

1. go module 이름 충돌 — module techai-ide가 다른 패키지에 import 될 경우 (github.com/kimjiwon/tgc/internal/*를 참조). 현재 techai-ide/go.mod는 독립 모듈이고 cli 쪽 internal을 import 하지 않음 → 문제 없음.
2. Wails 바인딩 재생성 — wailsjs/go/main/App.d.ts가 TECHAI 쪽 타입을 참조할 수 있음 → wails dev 한 번 돌리면 재생성됨. 5분.
3. knowledge embed 리팩터링 — //go:embed knowledge/docs 경로가 techai 의 bxm 포함이라면 수동 교정 필요. 10~15분.
